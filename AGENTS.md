# Repository conventions

## Pull requests

- 모든 PR은 [`.github/pull_request_template.md`](.github/pull_request_template.md)를 사용한다.
- 템플릿 섹션을 삭제하지 않는다. 해당 사항이 없으면 `없음` 또는 실행하지 않은 이유를 적는다.
- 변경 배경, 사용자·개발자 영향, 호환성·마이그레이션, 실제 검증 결과와 남은 위험을 구체적으로 작성한다.
- 테스트나 빌드는 실제로 실행한 항목만 완료 표시하며 실행 명령과 결과를 함께 남긴다.
- 스키마 변경 PR은 기존 데이터 호환성, 승격·마이그레이션 방식, 실제 데이터셋 검증 결과를 반드시 포함한다.
- PR을 생성하거나 수정할 때 자동 생성 본문으로 템플릿을 우회하지 말고 저장소 템플릿을 기준으로 본문을 작성한다.
