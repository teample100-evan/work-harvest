# 체크포인트 기밀 정책

Work Harvest의 기밀 수준은 표시용 메타데이터가 아니라 파생 출력과 알림을 통제하는 정책이다.

## 수준별 처리

| 수준 | 로컬 상세 화면 | 성과 노트 | 알림·메뉴바 |
| --- | --- | --- | --- |
| `normal` | 전체 표시 | 전체 포함 | 업무 제목과 체크포인트 ID 표시 가능 |
| `sensitive` | 전체 표시, 민감 배지 | 제목·요약·결과와 검증 상태만 포함. 활동 상세, 결정, 명령, 파일, URL, Git, 근거 참조는 생략 | 업무 제목만 표시 가능. 기록 본문과 근거는 표시하지 않음 |
| `restricted` | 사용자가 업무 상세를 열었을 때만 전체 표시 | 기본적으로 완전 제외 | 새 체크포인트 알림과 마지막 기록 메뉴에서 제외 |

## 파생 문서

- 민감 체크포인트가 하나라도 포함된 성과 노트는 front matter의 `confidentiality`를 `sensitive`로 기록한다.
- 제한 체크포인트는 성과 노트의 체크포인트 ID, 작업일, 제목, 요약, 근거 어디에도 포함하지 않는다.
- 제한 체크포인트도 source revision 검증 대상에는 포함한다. 검토 후 원본이 추가·변경되면 보고서 저장을 중단한다.
- 보고서 생성 화면은 포함·세부 생략·제외 수를 저장 전에 표시한다.

## 운영 원칙

- 기밀 수준을 낮추는 작업은 자동화하지 않는다.
- 내보내기나 팀 공유 기능을 추가할 때도 이 정책을 기본값으로 상속한다.
- 사용자가 편집한 성과 노트 Markdown은 검토된 원문 그대로 저장하되, 제한 체크포인트 원문을 자동으로 다시 삽입하지 않는다.
