# 주간 성과보고서

데스크톱 앱의 사이드바 `주간 보고서`에서 월요일부터 일요일까지의 기본 범위를 열 수 있다. 날짜 탐색에서 특정 날짜를 선택한 상태라면 그 날짜가 속한 주가 기본값이 된다.

## 집계 기준

- 데스크톱 앱의 기본 범위는 `company`다. `personal`, `unclassified`, `all`로 바꿀 수 있다.
- `reporting.mode: primary` 업무만 기본 포함한다.
- `reporting.mode: supporting` 업무는 `보조 활동 포함`을 켰을 때만 포함한다.
- `reporting.mode: excluded` 업무는 항상 제외한다.
- 체크포인트의 `work_period.start`와 `work_period.end`가 선택 기간과 겹치면 포함한다.
- 작업 기간이 모두 `null`인 체크포인트는 집계에서 제외하고 `기간 미확인` 수치로 표시한다.
- correction이 가리키는 원본 체크포인트는 성과 집계에서 대체하고 정정된 유효 기록만 사용한다.
- 포함된 체크포인트를 업무 항목별로 묶고 최신 결과와 영향을 최대 3개까지 선별한다.
- Git 본문에는 업무당 최신 PR 1개를 표시하고, PR이 없을 때만 최신 Git 기준점을 사용한다. 전체 커밋·명령·파일은 근거 기록에서 확인한다.
- 검증은 유형별 최신 상태를 먼저 선별한 뒤 업무당 종합 상태 1줄로 압축하고, 확인이 필요한 결과를 우선한다. 명령 전문은 본문에 표시하지 않는다.
- 위험과 다음 작업은 과거 체크포인트가 아니라 현재 Context에서만 가져오며 각각 최대 10개를 표시한다.
- 앱이 임의의 테스트 명령을 백그라운드에서 실행하지는 않는다. 보고서는 구조화된 기록에 남은 결과만 사용한다.

## 기밀정보 처리

- `normal`: 전체 구조화 근거를 포함한다.
- `sensitive`: 요약과 결과·검증 상태만 포함하고 활동 세부, 명령, Git, 링크와 파일 근거를 생략한다.
- `restricted`: 보고서에서 완전히 제외한다.

민감 체크포인트가 하나라도 포함되면 보고서 front matter의 `confidentiality`는 `sensitive`가 된다. 저장 전 Markdown 편집으로 이 값을 `normal`로 낮출 수 없다.

## 저장과 충돌 방지

기본 경로는 다음과 같다.

```text
reports/weekly/<시작일>_to_<종료일>_<scope>.md
```

`scope: all`일 때만 이전과 같은 `<시작일>_to_<종료일>.md` 경로를 사용한다. 회사 업무를 기본으로 검토하는 앱에서는 예를 들어 `20260713_to_20260719_company.md`가 된다.

앱은 미리보기 시점의 업무·Context·체크포인트 revision을 함께 보관한다. 같은 경로의 기존 보고서가 있으면 기존 내용과 새 초안의 변경을 보여주고, 저장 시 덮어쓰기 여부를 다시 확인한다. 기존 보고서나 집계 원본이 검토 중 바뀌면 덮어쓰지 않고 최신 원본으로 다시 검토하도록 안내한다.
